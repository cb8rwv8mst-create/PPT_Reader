# 分工文档

## 项目概述

AI PPT 讲解员：用户上传 .pptx 文件，AI 自动生成中文讲解稿并合成语音。前端 React、后端 Spring Boot。

## 分工总览

| 成员 | 模块 | 一句话 |
|---|---|---|
| A | 后端：PPT 解析 + REST API | PPTX 文件 → 结构化数据 → 接口给前端 |
| B | 后端：AI 讲解 + TTS 语音 | 幻灯片文本 → DeepSeek 生成讲解稿 → Edge TTS 生成 MP3 |
| C | 前端：React 页面 | 文件上传、幻灯片展示、讲解文本展示、音频播放 |

---

## 成员 A：PPT 解析 + REST API

### 要做的事

1. **添加 Maven 依赖**：`spring-boot-starter-web`、Apache POI (`poi-ooxml`)
2. **PPTX 解析**：用 Apache POI XSLF 读取 .pptx 文件，提取每页的：
   - 标题（title）
   - 正文（段落文本）
   - 备注（notes）
3. **数据模型**：创建 `Slide.java`、`Narration.java`
4. **文件上传接口**：`POST /api/ppt/upload`，接收 multipart 文件，保存到本地临时目录，返回一个任务 ID + 解析后的幻灯片列表
5. **幻灯片查询接口**：`GET /api/ppt/{id}/slides`
6. **静态文件访问**：使上传的 PPTX 和生成的音频可访问
7. **配置 CORS**：允许前端 `localhost:5173` 跨域访问

### 关键 Java 类

```
controller/PptController.java       — 上传、查询接口
service/PptParserService.java       — POI 解析逻辑
model/Slide.java                    — 幻灯片数据类
model/PptTask.java                  — 任务状态类（id、文件路径、创建时间）
```

### POI 解析示例

```java
XMLSlideShow ppt = new XMLSlideShow(new FileInputStream(file));
List<XSLFSlide> slides = ppt.getSlides();
for (XSLFSlide slide : slides) {
    String title = slide.getTitle();                    // 标题
    String content = slide.getTextContent();            // 正文
    String notes = slide.getNotes().getTextBody();      // 备注
}
```

## 成员 B：AI 讲解 + TTS 语音

### 要做的事

1. **添加依赖**：可能需要 `spring-boot-starter-webflux`（用于 HTTP 调用）、Jackson
2. **DeepSeek 集成**：
   - 调用 DeepSeek API（`https://api.deepseek.com/chat/completions`）
   - 构造 Prompt：把幻灯片文本传给模型，让其生成自然流畅的中文讲解稿
   - API Key 从 `application.properties` 或环境变量读取
3. **Edge TTS 集成**：
   - 调用 Microsoft Edge TTS 服务，将讲解稿文本转为 .mp3 音频
   - 保存音频文件到本地目录
4. **讲解生成接口**：`POST /api/ppt/{id}/narrate`
   - 接收幻灯片内容 → 调 DeepSeek → 返回讲解稿文本 + 每页讲解分段
5. **语音合成接口**：`GET /api/ppt/{id}/audio`
   - 返回生成的 .mp3 音频文件

### 关键 Java 类

```
service/DeepSeekService.java        — DeepSeek API 调用
service/EdgeTTSService.java         — Edge TTS 语音合成
model/Narration.java                — 讲解稿数据类
```

### DeepSeek Prompt 参考

```
你是一个专业的中文演讲者。请根据以下PPT幻灯片内容，生成一段自然流畅的讲解稿。
要求：口语化、有逻辑衔接、每页讲解用 [SLIDE_X] 标记分隔。

PPT内容：
[幻灯片文本...]
```

### Edge TTS 调用方式

Edge TTS 底层是 Microsoft Speech Service 的 WebSocket 接口。可以直接用 Java HTTP 客户端发请求，或者用 `ProcessBuilder` 调 Python 的 `edge-tts` 命令行工具：

```bash
edge-tts --voice zh-CN-XiaoxiaoNeural --text "讲解文本" --write-media output.mp3
```

## 成员 C：React 前端

### 要做的事

1. **初始化项目**：在 `frontend/` 下用 `npm create vite@latest . -- --template react-ts` 创建
2. **上传页（首页）**：
   - 拖拽或点击上传 .pptx 文件
   - 上传后调 `POST /api/ppt/upload`，拿到任务 ID
   - 跳转到讲解页
3. **讲解页**：
   - 左侧：幻灯片缩略图列表，点击切换
   - 中间：当前幻灯片的内容（标题、正文、备注）
   - 右侧：AI 生成的讲解稿，当前页对应段落高亮
4. **音频播放器**（页面底部）：
   - 播放/暂停、进度条、音量
   - 播到哪句高亮哪句（可选，加分项）
5. **状态管理**：用 React 内置 `useState`/`useEffect` 即可，不需要额外库

### 关键文件

## 三人接口约定（重要）

A、B、C 三人按这几个接口对接，互不阻塞：

### 1. 上传接口（A 提供，C 调用）

```
POST /api/ppt/upload
Content-Type: multipart/form-data
参数: file (PPTX文件)

→ 返回:
{
  "id": "abc123",            // 任务ID
  "slides": [
    { "index": 0, "title": "...", "content": "...", "notes": "..." },
    ...
  ]
}
```

### 2. 幻灯片查询（A 提供，B 和 C 调用）

```
GET /api/ppt/{id}/slides

→ 返回: 同上 slides 数组
```

### 3. 讲解生成（B 提供，C 调用）

```
POST /api/ppt/{id}/narrate
Content-Type: application/json
Body: { "slides": [ { "index": 0, "title": "...", "content": "..." }, ... ] }

→ 返回:
{
  "fullScript": "完整讲解稿文本...",
  "slideScripts": [ "第1页讲解...", "第2页讲解...", ... ]
}
```

### 4. 音频获取（B 提供，C 调用）

```
GET /api/ppt/{id}/audio

→ 返回: audio/mpeg 二进制流
```

## 协作阶段

1. **第零步**：三人一起搭好开发环境（JDK、Maven、Node.js、Git）
2. **第一步**：A 和 B 各自拉分支开发后端，C 用 Mock 数据开发前端，互不阻塞
3. **第二步**：A 的接口就绪后，C 联调上传和幻灯片展示
4. **第三步**：B 的接口就绪后，C 联调讲解生成和音频播放
5. **第四步**：三人一起整体联调、修 bug

> C 建议先用假数据写前端 UI，不依赖 A 和 B 的接口就能跑起来。
